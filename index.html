Pase de Dato
Se denomina pase de datos a la asignación de valores a una varible. Esta asignación de datos se puede realizar de dos formas: por valor o referencia. El pase de datos por valor es cuando se le asigna a la variable un valor que no guarda relación con la variable que se lo asignó por lo que al modificar el valor de esta la que se lo asignó no sufre cabios (Si es un tipo de dato primitivo es por valor). Por otro lado en el pase de datos por referencia la variable guarda un vinculo (puntero o referencia) con la variable que le asignó el valor y si se modifica esta también se modificará la otra (Si es un dato no primitivo como funciones, objetos u arrays es por referencia).
Pase de datos por valor
let a = "Valor inicial";
let b = a;
a = "Valor cambiado";
//En este punto b sigue con: "Valor inicial".
Pase de datos por referencia
const a = ["Valor inicial"];
const b = arrayA;
a.push("Otro valor");
//En este punto b contiene: ['Valor inicial', 'Otro valor']

Programación Orientada a Objetos
En JavaScript, la programación orientada a objetos (POO) se lleva a cabo utilizando objetos y funciones. Aunque JavaScript es un lenguaje basado en prototipos puede implementar las caracteristicas básicas de la POO tales como la encapsulación, la herencia y el polimorfismo.
Prototipos
Los prototipos en JavaScript son un mecanismo de herencia que permite a los objetos compartir propiedades y métodos. Cada objeto en JavaScript tiene una propiedad interna llamada "prototype" (prototipo), que es otro objeto. Este objeto prototipo puede contener propiedades y métodos que se heredarán por el objeto original. El concepto de prototipos es fundamental en JavaScript, ya que es un lenguaje basado en prototipos. Esto significa que en lugar de utilizar una estructura de clases tradicional, JavaScript utiliza prototipos para implementar la herencia y el polimorfismo. Cuando se crea un nuevo objeto, este objeto hereda las propiedades y métodos de su objeto prototipo. Si se intenta acceder a una propiedad o método que no está definida en el objeto original, JavaScript buscará automáticamente en el objeto prototipo y, si no se encuentra, continuará subiendo por la cadena de prototipos hasta encontrar la propiedad o método deseado. En resumen, los prototipos en JavaScript son una forma de compartir comportamientos y atributos entre objetos, lo que facilita la reutilización de código y la creación de estructuras de datos más flexibles y dinámicas.
Métodos prototipos
Los metodos prototipos no estan instanciados en los objetos, solo se usan o se instancian en los objetos que los necesiten. Tomando como ejemplo el objeto Persona que solo tiene un atributo nombre se le va asignar un método prototipo llamado saludar.
function Persona(nombre) {
 this.nombre = nombre;
}
Persona.prototype.saludar = function () {
 return `${this.nombre} dice hola!`;
}

Clases
Las clases en JavaScript son una estructura de datos introducida en ES6. Las clases se utilizan para agrupar variables y funciones en un elemento padre relacionado. En términos simples, una clase es un modelo que define un conjunto de propiedades y comportamientos que pueden ser compartidos por múltiples objetos (instancias). Tienen un método especial llamado constructor, es constructor se llama automáticamente cuando se crea una nueva instancia de la clase. Los parámetros entre paréntesis en el constructor son opcionales y se utilizan para inicializar las propiedades de la clase. Los metodos dentro de un objeto definido con class se comvierten automaticamente en funciones protoripo (Persona.prototype.saludar).
Ejemplo de uso
class Persona {
 constructor(name, age) {
  this.name = name;
  this.age = age;
 }
 saludar = function () {
  return `${this.name} dice hola desde la clase Persona`;
 };
}

Herencia
La herencia en JavaScript es un concepto que nos permite hacer que un objeto herede los comportamientos de otro. Esto se logra a través de la cadena de prototipos, donde cada objeto tiene una propiedad privada llamada [[Prototype]] que mantiene un enlace a otro objeto llamado su prototipo. La herencia en JavaScript nos permite organizar clases utilizando una jerarquía, lo que nos permite escribir menos código y reutilizar características. Esto es especialmente útil en la creación de patrones de diseño y la construcción de aplicaciones más grandes y complejas. En resumen, la herencia nos permite crear nuevos objetos que heredan propiedades y métodos de otros objetos, lo que nos permite reutilizar código y estructurar nuestro código de manera más eficiente y organizada.
Ejemplo de uso
class Estudiante extends Persona {
 //# denota atributos privados (no accesibles fuera de la clase)
 #notas = [];
 constructor(name, edad, notas = []) {
  //super accede al método constructor de la clase de quien hereda (Persona)
  super(name, edad);
  this.#notas = notas;
 }
 saludar = function () {
  return `${this.name} dice hola desde la clase Estudiante`;
 };
 //Métodos get y set para acceder a atributos privados
 set _notas(nota) { this.#notas.push(nota); }
 get _notas() { return this.#notas; }
}
Instancia la clase Estudiante

Query Selectors
QuerySelector es un selector de elementos del DOM que devuelve el primer elemento del documento que coincida con el grupo especificado de selectores (para buscar utilizan un recorrido primero en profundidad pre ordenado de los nodos del documento). Para selecciones multiples se puede usar querySelectorAll que funciona igual pero devuelve una lista con todos los elementos que coincidan con el selector.
Ejemplos de uso
document.querySelector('p')); //por tipo de etiqueta
document.querySelector('#identificador')); //por id
document.querySelector('.clase')); //por nombre de clase
document.querySelector('.clase1 .clase2')); //por nombre de clase anidada
//Aquí, será retornado el primer elemento <input name="login" /> dentro de <div class="user-panel main">.
document.querySelector("div.user-panel.main input[name='login']")
document.querySelectorAll('.text-danger');
document.querySelectorAll('.container .text-danger');

Document Fragment
Document.createDocumentFragment() Crea un nuevo DocumentFragment vacío, dentro del cual un nodo del DOM puede ser adicionado para construir un nuevo arbol DOM fuera de pantalla, sobre este DocumentFragment se realizan todas la operaciones de inserción requeridas y después se agrega este al DOM lo que es una sola operación de refrescamiento de la pantalla. Esto hace que si hay que hacer muchas inserciones no se satura el DOM, porque cada vez que se agrega algun elemento este se renderiza.
HTML Templates
El elemento HTML <template> es un mecanismo para mantener el contenido HTML del lado del cliente que no se renderiza cuando se carga una página, pero que posteriormente puede ser instanciado durante el tiempo de ejecución empleando JavaScript. El analizador procesa el contenido del elemento <template> durante la carga de la página, pero sólo lo hace para asegurar que esos contenidos son válidos, sin embargo, estos contenidos del elemento no se renderizan.

Callbacks
Las funciones callbacks en JavaScript se utilizan para manejar la ejecución de código de manera asíncrona y controlar el flujo de ejecución.
  • Ejecución asíncrona: Los callbacks permiten que una función se ejecute después de que se complete una tarea, como una solicitud de red o una operación de entrada/salida. Esto es especialmente útil en JavaScript, ya que es un lenguaje de programación de un solo hilo, lo que significa que solo puede ejecutar una tarea a la vez.
  • Control del flujo de ejecución: Los callbacks permiten que el código se ejecute en un orden específico, incluso si las tareas se completan en un orden diferente. Esto es útil cuando se necesita que ciertas tareas se completen antes de que se ejecute otra tarea.
  • Flexibilidad: Los callbacks permiten que las funciones sean más flexibles y reutilizables. Al pasar una función como un argumento a otra función, se puede personalizar el comportamiento de la función principal de acuerdo con las necesidades específicas.
En resumen, los callbacks en JavaScript se utilizan para manejar la ejecución asíncrona, controlar el flujo de ejecución y aumentar la flexibilidad y la reutilización de las funciones.

Módulos
En JavaScript, los módulos son un conjunto de funciones o código relacionado que se agrupan en un archivo. Estos módulos permiten dividir el código en partes más manejables, lo que facilita la mantenibilidad del código base. Los módulos se pueden importar entre sí.
Para utilizar módulos en JavaScript, hay que utilizar las sintaxis de importación y exportación. Si se va a importar un modulo hay que ir al html y declarar el script que contendrá la importacion como type="module" y si es *.js el script q se va a importa hay que agregar la extencion o da error la importacion. Para que este proceso de importación funcione correctamente se requiere que la página este corriendo en un servidor porque si se habre la página directamente desde el file da error de CORS (Cross-Origin Resource Sharing), que se produce cuando un navegador web bloquea una solicitud que proviene de un origen diferente al del sitio web actual. Cuando intentas importar un módulo JavaScript utilizando la sintaxis import en un archivo HTML, el navegador trata de obtener el módulo desde una URL diferente al archivo HTML actual. Esto se considera una solicitud entre orígenes diferentes y desencadena un error de CORS. Si el servidor no está configurado correctamente para permitir solicitudes desde orígenes diferentes también se producirá pero esto no suele suceder comúnmente.
Por ejemplo, para exportar un arreglo llamado "frutas" (Se pueden exportar funciones, clases, var, let y const) desde un archivo llamado "modulo1.js", se utiliza la siguiente sintaxis:
export const frutas = ["🍇", "🍉", "🍍", "🥭", "🍎"];
Y para importar el arreglo "frutas" en otro archivo, se utilizar la siguiente sintaxis (como la extención del modulo1 es *.js hay que ponerlo explícitamente):
import { frutas } from "./modules/modulo1.js";
console.log(frutas);
El arreglo se puede utilizar inmediatamente después de importarlo. Se pueden usar alias para el los nombres de los elementos que se van a exportar con vistas a evitar colisiones de nombres de haberlas.
import { frutas as otroNombre} from "./modules/modulo1.js";
console.log(otroNombre);
